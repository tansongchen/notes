关于工具选择的思考

  1. 以系统为纲：先有科学学习系统/自我管理系统，然后再将它的每个部分对应到工具；
  2. 狐狸思维：每个工具都有其擅长的部分，不能期待一个大而全的工具扛下所有，而是要多个小而美的工具互相配合；
  3. 联结主义：由于上一条，
    1. 是否支持插件的编写
    2. 对于网络优先的应用，是否支持 API
    3. 对于本地优先的应用，是否支持 Automator, macOS 内链
  4. 定期迭代：选定一个工具之后，应该先不加怀疑地使用一段时间，可能每隔 2 年需要重新审视一下新的工具生态，看是否需要作出改变

附：小能熊工具方法论：

  1. 梳理需求
    1. 了解自己到底需要什么
  2. 主题研究
    1. 市面上有哪些解决方案
    2. 各个解决方案，各有何利弊？
  3. 克制极致
    1. 根据当下需求以及对解决方案的理解
    2. 选择一个好的，先用起来
    3. 高强度的实践和思考
  4. 迭代，而非完美
    1. 在做中学，在用中优化
